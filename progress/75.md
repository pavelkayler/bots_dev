# 75 - Signal bot follow-tail baseline + providers endpoint checks

Date: 2026-03-07

## 1) Goal
Add operational visibility and a strict follow-tail bootstrap flow for the signal bot page:
- frontend view of Bybit/CoinGlass endpoint availability
- follow-tail baseline receive flow (start date to now, no mode selector)
- minute-interval baseline request for optimizer compatibility
- tracking tied to recorder mode

## 2) Backend changes

### 2.1 Provider capabilities service
Files:
- `backend/src/api/providerCapabilities.ts`
- `backend/src/api/http.ts`

Added endpoint:
- `GET /api/providers/capabilities`

Checks returned:
- Bybit server time
- Bybit linear tickers
- Bybit OI history (5m endpoint availability)
- Bybit funding history
- CoinGlass 1m OI history capability (with key/plan/rate-limit classification)

Response includes:
- per-check status, availability, message, latency
- summary counters and required-endpoint readiness

### 3.2 Signal follow-tail dataset card
File:
- `frontend/src/pages/signalBot/SignalFollowTailDatasetCard.tsx`

Behavior:
- only follow-tail style bootstrap input
  - `Universe`
  - `Start date`
- fixed receive window:
  - `start -> now`
- receive request uses interval `"1"` (minute baseline)
- progress polling + cancel support

### 3.3 Signal bot live tab baseline gating
File:
- `frontend/src/pages/signalBot/SignalBotPage.tsx`

Added:
- latest dataset baseline checks for required signal-bot metrics
- explicit missing metrics list
- tracking lock while baseline incomplete
- tracking toggles recorder mode:
  - enabled -> `record_only`
  - disabled -> `off`
