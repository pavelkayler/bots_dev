# Test rollout report

## How to run tests

### Backend
- Install deps: `npm --prefix backend install`
- Run tests: `npm --prefix backend test`
- Build check: `npm --prefix backend run build`

### Frontend
- Build check: `npm --prefix frontend run build`

No frontend test harness was added because the frontend did not already include Vitest/RTL and frontend tests were scoped as optional when absent.

## Coverage summary

### A) PaperBroker unit coverage
Implemented `backend/src/tests/paperBroker.test.ts` covering:
- Quantity computation (`margin * leverage / entryPrice`).
- Entry fill conditions for LONG/SHORT and no-fill cases.
- TP/SL derivation from ROI and leverage.
- TP/SL close event emission and realized PnL sign behavior.
- Maker fee accounting on both entry and exit.
- Entry expiry behavior (`ORDER_EXPIRED`, no position open).
- Rearm delay preventing immediate re-entry.
- `stopAll({ closeOpenPositions: false })` clearing pending/open state without adding closed-trade stats.

### B) SignalEngine unit coverage
Implemented `backend/src/tests/signalEngine.test.ts` covering:
- `no_refs` reason when refs are missing.
- Threshold pass returning an `ok_*` decision.
- Funding-sign gating behavior with `requireFundingSign=true`.
- `cooldownActive` returning `cooldown` and suppressing signal.

### C) Optimizer replay invariants
Implemented `backend/src/tests/optimizerRunner.test.ts` for helper-level invariants:
- Close-only replay events built at candle close timestamps.
- OI/OIV mapping (`openInterestValue = oi * close`, and `0` when missing/zero).
- No turnover/volume fallback usage in replay helper outputs (indirectly asserted by OIV-zero behavior when OI absent).
- Funding sample lookup semantics (`last-known <= ts`) and funding-direction decision gating.

To support deterministic helper tests, extracted and exported pure helpers from `runner.ts`:
- `fundingRateAtTs`
- `createReplayEventsFromCacheRows`
- `buildCandidateKey`

### D) rememberNegatives / blacklist behavior
Implemented `backend/src/tests/optimizerRunner.test.ts` coverage for:
- Candidate key determinism.
- Blacklist persistence round-trip using temp cwd-backed files.
- Repeated-key skip semantics within evaluation loops (set-membership skip logic), plus run metadata persistence (`runIndex`).

## Weak spots found

1. **Mixed test frameworks in tree**
   - Existing `backend/src/tests/invariants.test.ts` uses `node:test`; Vitest initially tried to execute it and failed collection.
   - Mitigation added: Vitest include/exclude config to only target Vitest suites.

2. **Optimizer cache paths were cwd-bound at module load**
   - `CACHE_DIR` and funding path constants were previously computed once on import, making test isolation harder when cwd changes.
   - Minimal seam added: path resolution moved to functions (`getCacheDir`, `getFundingCacheDir`) so tests can isolate filesystem state deterministically.

3. **End-to-end optimizer negative-blacklist flow remains difficult to make fully deterministic in short unit tests**
   - Candidate evaluation depends on replay timing, reference buckets, and strategy decisions.
   - Current tests cover deterministic building blocks and blacklist persistence/skip semantics directly, but full-path scenario assertions can still be brittle without additional seams.

## Minimal follow-up improvements (not implemented)

1. Add a small exported pure evaluator function for blacklist decision (`shouldSkipCandidate`) to remove duplication and improve test readability.
2. Add an optional in-memory filesystem adapter seam to optimizer blacklist/cache readers for even faster and cleaner tests.
3. Consider gradually migrating legacy `node:test` suites to Vitest for one consistent harness and easier CI config.
