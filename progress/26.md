1) What changed per file
- `frontend/src/features/datasetTarget/ui/DatasetTargetCard.tsx`
  - Hardened all range/universe `onChange` handlers to safely read event values (`e?.currentTarget?.value ?? ""`) before state updates.
  - When switching range mode back to preset, now ensures `preset` is always valid (fallback to first preset / `24h`).
- `backend/src/optimizer/runner.ts`
  - Exported `OptimizerParams` and added `simulateCandidateTrades(args, params)` helper.
  - Added support for fixed candidate params in `runOptimizationCore` (`fixedParams`) and a candidate completion hook to capture TP/SL close events and candidate stats.
- `backend/src/api/http.ts`
  - Added local-only endpoint `GET /api/optimizer/jobs/:jobId/trades/export?rank=<rank>&format=json`.
  - Endpoint resolves candidate by rank from sorted results, re-simulates that one candidate with existing run payload settings, and returns run payload + params + direction + trades + summary.
- `frontend/src/features/optimizer/api/optimizerApi.ts`
  - Added `getJobTradesExportUrl(jobId, rank)` helper for the new endpoint.
- `frontend/src/pages/optimizer/OptimizerPage.tsx`
  - Added per-row `Export trades` action in optimizer results table.
  - Renamed `Copy to settings` buttons to `Copy`.
  - Made “Hide negative netPnl” pure UI filtering only; loop start now always sends `excludeNegative: false` while still sending `rememberNegatives`.
  - Moved “Hide negative netPnl” control to results table header and kept it enabled during active runs.
  - Rearranged optimization form layout so margin/leverage (+ fee/slippage) are on a second row.

2) Why (mapping to items 1–5)
- (1) Prevent DatasetTargetCard null event crashes and preset invalid state when toggling manual/preset.
- (2) Provide targeted trade export for a specific ranked result row, including run payload and candidate params, without bloating normal results payload.
- (3) Make hide-negative behavior UI-only and independently toggleable during running loops; keep remember-negatives backend blacklist behavior.
- (4) Apply requested optimizer form row layout change.
- (5) Apply action label rename to `Copy`.

3) How to verify locally
- Build checks:
  - `npm --prefix backend run build`
  - `npm --prefix frontend run build`
- Dataset target crash fix:
  - Open Optimizer page Dataset Target card.
  - Toggle Range mode `Manual -> Preset -> Manual` repeatedly and change values quickly.
  - Confirm no null `currentTarget.value` crash.
- Hide-negative behavior:
  - Start loop.
  - Toggle `Hide negative netPnl` in table header while results stream updates.
  - Confirm negatives hide/show instantly without disabling checkbox.
  - Start a new loop with `Remember negatives` enabled and confirm previously negative settings are still skipped across runs.
- Export trades:
  - In results table click `Export trades` for any row.
  - Verify new tab opens `/api/optimizer/jobs/<jobId>/trades/export?rank=<rank>&format=json`.
  - Confirm payload includes: `jobId`, `rank`, `runPayload`, `params`, `directionMode`, `trades`, `summary`.

4) Known limitations
- Trade export re-simulates only the selected candidate at export time (not persisted original event stream).
- Trades export endpoint is local-only by request policy.
