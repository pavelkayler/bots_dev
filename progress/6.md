## What was broken
- Starting optimizer loop returned 400 (`optimizer_loop_start_failed`) even with selected history row and cache present.

## Exact root cause
- `/api/optimizer/run` partially migrated to dataset-history mode but still used stale `datasetTarget` variables (`target`, `resolvedRange`, `symbols`) that were no longer defined in this path.
- The resulting runtime failure made internal `app.inject("/api/optimizer/run")` fail in `startLoopJob()`, so `optimizerLoopState.lastJobId` was never set and `/api/optimizer/loop/start` returned 400.
