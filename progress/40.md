1) What changed
- Removed the effect in `OptimizerPage` that was forcing `sortKey` to `valPnlPerTrade` (and `sortDir` to `desc`) when `val pnl/trade > 0` filter was enabled.
- Removed the redundant "Sort: val pnl/trade" button from the optimizer table controls.

2) Why
- The forced sort effect caused user-selected `netPnl` sorting to revert immediately whenever `filterValPnlPerTradePos` was enabled, so header-driven sorting did not persist.

3) How to verify
- Enable the `val pnl/trade > 0` filter.
- Click the `netPnl` column header and confirm sorting remains on `netPnl` (no immediate revert).
- Click the `val pnl/trade` header and confirm sorting switches to that column.
- Confirm there is no `Sort: val pnl/trade` button in the controls row.
- Run `npm --prefix frontend run build` to confirm the frontend still builds.
