# Stage 5 - Recorder Universe (separate persistent selection)

## 1) Goal
Introduce a dedicated recorder universe configuration, separate from trading universe, so permanent recording can target its own symbol set.

## 2) Backend implementation
1. Added recorder universe store:
   - `backend/src/recorder/recorderUniverseStore.ts`
2. Persisted state shape:
   - `selectedId: string | null`
   - `symbols: string[]`
   - `updatedAtMs: number`
3. Storage path:
   - `backend/data/recorder/universe.json`
4. Resolution rules:
   - if recorder universe has symbols -> use them
   - if empty -> fallback to trading universe symbols from runtime config
5. API endpoints:
   - `GET /api/recorder/universe`
   - `POST /api/recorder/universe`
     - `{ selectedId }` resolves symbols from saved universe
     - `{ symbols: [] }` stores explicit symbols directly
6. WS hub integration:
   - recorder activation now uses `resolveRecorderSymbols()` instead of trading universe directly.

## 3) Tests
Added `backend/src/tests/recorderUniverseStore.test.ts`:
1. persists explicit symbol list with normalization/dedup
2. resolves symbols correctly when selecting a saved universe by id

## 4) Docs
Updated `docs/11_current_state.md` with recorder universe endpoints and separation from trading universe.
