# Stage 6 - Daily Chunked Recorder Storage

## 1) Goal
Move recorder storage from a single symbol file to daily chunked files suitable for append, copy, and later incremental merging.

## 2) Changes made
1. Updated `MinuteOiRecorder` storage layout:
   - old: `.../bybit_open_interest_1m/<SYMBOL>.jsonl`
   - new: `backend/data/recorder/bybit/open_interest_1m/<SYMBOL>/<YYYY-MM-DD>.jsonl`
2. Write behavior:
   - append-only line write to day chunk based on minute timestamp
3. Read behavior:
   - merges rows from all chunk files for symbol
   - keeps compatibility by also reading legacy single-file path if present
4. Last-ts bootstrapping:
   - when in-memory last timestamp is absent, recorder scans available files for latest line timestamp and resumes dedup from there.

## 3) Tests
Updated `backend/src/tests/minuteOiRecorder.test.ts`:
1. verifies chunk file creation for multiple days
2. verifies merged reads across day files
3. existing tests continue to verify boundary skipping and timestamp-merge behavior
