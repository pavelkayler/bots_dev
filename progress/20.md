1) What changed (per file)
- `backend/src/optimizer/runner.ts`
  - Updated the `replayInterval === "1"` cache replay branch to emit intrabar synthetic ticker events from 1m OHLC candles instead of a single close-only tick.
  - New 1m replay sequence now mirrors the non-1m branch structure:
    - `candleStart` open tick
    - `candleStart + 1` / `candleStart + 2` low-high or high-low ticks depending on candle direction
    - `tsClose` close tick
    - `kline_confirm` at `tsClose`
  - Added per-tick OI value calculation in the 1m branch:
    - `oiv = oiBase > 0 ? oiBase * markPriceForTick : 0`
    - sets both `openInterestValue` and `openInterest` to `oiv`
  - Kept 1m replay funding at `0`.
- `progress/20.md`
  - Added this report for the bugfix implementation, root cause, verification, and limitations.

2) Why (root cause + fix)
- Root cause:
  - 1m cache replay emitted only one ticker per candle at close time (`tsClose`), causing roughly 60-second gaps between ticks.
  - In cacheDataset replay, `medianTickIntervalSec` could remain `0`, so entry timeout stayed at randomized `timeoutSec` values that are often below 60s.
  - Result: orders placed but no intermediate ticks to trigger fill checks before timeout, so orders expired unfilled.
- Fix:
  - Emit synthetic intrabar ticks for 1m candles so PaperBroker receives multiple tick opportunities within each minute.
  - Keep a close tick and `kline_confirm` to stay consistent with non-1m replay semantics.
  - Compute OI value per emitted tick using each tick's mark price for consistency.

3) How to verify locally
- Build:
  - `npm --prefix backend run build`
  - `npm --prefix frontend run build`
- Functional replay verification:
  1. Run optimizer with `cacheDataset` over a range with populated 1m cache.
  2. Confirm replay emits ticker events at `candleStart`, `+1ms`, `+2ms`, and `tsClose` for ranged candles.
  3. Confirm each candle emits `kline_confirm` at `tsClose`.
  4. Compare run stats vs. previous behavior: `ordersFilled` should become non-zero for viable candidates and `ordersExpired` should no longer equal `ordersPlaced` universally.

4) Known limitations
- Replay remains bar-based and uses a synthetic intrabar ordering heuristic (open -> low/high -> close by candle direction), not true tick-level history.
- Fill quality still depends on candle OHLC granularity and timeout settings; this change improves tick cadence but does not reconstruct real microstructure.
