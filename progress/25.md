1) What changed
- Removed the "Reset" button from the Optimizer loop results toolbar in `frontend/src/pages/optimizer/OptimizerPage.tsx`.
- Kept `resetLoopResultsState()` unchanged so it is still used by `onStartLoop()` for automatic reset behavior.

2) Why
- The manual Reset control is not needed in the loop results UI.
- "Start Loop" already performs the correct auto-reset flow by calling `resetLoopResultsState()` before `startOptimizerLoop()`.

3) How to verify locally
- Run: `npm --prefix frontend run build`
- Open the Optimizer page and confirm the toolbar no longer shows a "Reset" button.
- Click "Start Loop" and verify loop results are still cleared at start (auto-reset remains intact).
