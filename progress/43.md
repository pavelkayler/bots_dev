1) .gitignore changes
- Kept broad runtime ignores for `backend/data/**` artifacts that are noisy for commits (sessions/checkpoints/history/tapes).
- Explicitly unignored cache subtrees so market cache files are now trackable:
  - `backend/data/cache/bybit_klines/**`
  - `backend/data/cache/bybit_open_interest/**`
  - `backend/data/cache/bybit_funding_history/**`
  - `backend/data/cache/manifests/**`
- `backend/data/sessions/**` and `backend/data/optimizer_checkpoints/**` remain ignored.

2) Paper settings ms->sec changes + copy mapping
- Updated Paper settings UI field from `rearmDelayMs` to `rearmSec`.
- UI now derives initial seconds from existing config milliseconds via rounding.
- Save/apply path converts seconds back to milliseconds for backend config payload (`rearmDelayMs = rearmSec * 1000`).
- Optimizer "Copy" now writes `paper.rearmSec` so copied settings align with seconds-based Paper UI.
- Pending patch ingestion in config supports both `rearmSec` and legacy ms keys for compatibility.

3) Navigation fix summary
- Removed `v7_startTransition` usage from `RouterProvider`.
- This avoids transition-based route commits being starved while optimizer page is producing frequent live updates, so header navigation now rerenders immediately to the destination page.

4) Results dedup fix
- Backend result `rowId` now uses stable candidate identity per job (`jobId + candidateKey`) instead of including run id, preventing repeat identity churn.
- Frontend loop results now aggregate rows by `candidateKey` across loop runs before rendering.
- Aggregation merges/sums core counters and pnl metrics (`netPnl`, `trades`, train/val totals, orders, long/short totals), then recomputes derived rates (`winRatePct`, train/val win rates, expectancy, valPnlPerTrade, side win rates).
- Result: loop runs with `runsCount > 1` render one combined row per candidate instead of triplicate rows.

5) How to verify locally
- Gitignore:
  - create or touch files in the cache folders above; they should appear as trackable in `git status`.
  - create files in `backend/data/sessions` / `backend/data/optimizer_checkpoints`; they should stay ignored.
- Paper settings:
  - open Config panel → Paper settings, confirm `rearmSec` field (no ms label).
  - save config and ensure backend receives/stores milliseconds (no runtime errors).
  - click optimizer row Copy and confirm Paper `rearmSec` receives expected seconds.
- Navigation:
  - start optimizer loop, click another header page; content should switch immediately with route change.
- Duplicates:
  - run loop with `runsCount=3`; table should show merged candidate rows instead of triple identical entries.
