# Stage 7 - Timestamp-based Merge Expansion

## 1) Goal
Ensure metric enrichment merges extend existing timestamped rows instead of replacing whole records.

## 2) Changes made
1. Added generic timestamp merge helpers:
   - `backend/src/dataset/timestampMerge.ts`
   - `upsertTimestampFields(...)`
   - `mergeFieldLayerByTimestamp(...)`
2. Applied helper in receive-data pipeline:
   - `backend/src/dataset/receiveDataStore.ts`
   - kline upsert now extends existing row fields at `startMs` instead of full replacement.

## 3) Why this matters
- Previously, re-upserting kline rows by timestamp could replace existing row objects and risk dropping already-attached metric fields.
- Now partial updates preserve prior fields (including future metric layers) and only update provided fields.

## 4) Tests
Added `backend/src/tests/timestampMerge.test.ts`:
1. verifies `upsertTimestampFields` preserves existing fields while updating incoming ones
2. verifies `mergeFieldLayerByTimestamp` extends only matching timestamps

## 5) Docs
Updated `docs/18_stability.md` with timestamp-safe merge behavior.
