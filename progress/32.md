1) What changed per file
- `backend/src/optimizer/runner.ts`
  - Added side win counters (`longsWins`, `shortsWins`) during TP/SL closes.
  - Added computed side winrate fields to each optimizer result row: `longsWinRatePct`, `shortsWinRatePct`.
  - Extended optimizer metric sort key union with side winrate keys.
- `frontend/src/features/optimizer/api/optimizerApi.ts`
  - Extended `OptimizationResult` with optional `longsWinRatePct` and `shortsWinRatePct`.
  - Extended `OptimizerSortKeyExtended` to include `pnlPerTrade`, `longsWinRatePct`, and `shortsWinRatePct`.
- `frontend/src/pages/optimizer/OptimizerPage.tsx`
  - Added derived sort/read support for `pnlPerTrade` (`netPnl / trades`, zero-safe when trades <= 0).
  - Inserted `pnl/trades` as the first optimizer results column and rendered it with fixed 4-decimal safe formatting.
  - Updated Longs/Shorts cell format to `count / pnl / winrate` using safe formatters.
  - Added normalization defaults for incoming side winrate fields.
  - Added side winrate keys into supported local sort keys and single-run fetch-sort list.
- `docs/17_optimizer.md`
  - Added deferred planning note for optional CoinGlass provider integration.

2) Why
- Added `pnl/trades` to make net efficiency per closed trade visible and sortable at a glance.
- Added backend-computed side winrates so Longs/Shorts columns can show `count / pnl / winrate` from authoritative simulation output.
- Recorded CoinGlass integration intent (deferred) to capture 1m OI/funding gap strategy without implementing it now.

3) How to verify locally
- Build backend: `npm --prefix backend run build`
- Build frontend: `npm --prefix frontend run build`
- Run optimizer UI and confirm:
  - First table column is `pnl/trades`.
  - Longs/Shorts render as `count / pnl / winrate`.
  - Sorting works for `pnl/trades` and side metrics (including side winrate keys).

4) Known limitations
- Single-run backend sort endpoint may not expose every UI local sort key uniformly for paginated server-side ordering in all combinations.
- `pnl/trades` is derived client-side and not persisted as a backend field.
