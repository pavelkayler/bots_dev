# Stage 3 - Market Data Recorder Foundation

## 1) Problem addressed
The repository had no dedicated recorder subsystem for collecting minute-level OI points that historical Bybit OI does not provide directly between 5-minute boundaries.

## 2) What was implemented
1. Added recorder foundation module:
   - `backend/src/recorder/MinuteOiRecorder.ts`
   - `backend/src/recorder/recorderStore.ts`
2. Recorder behavior:
   - accepts Bybit WS ticker OI points
   - rounds timestamps to minute
   - records only intermediate minutes (skips 5-minute boundary timestamps)
   - writes append-only JSONL per symbol under:
     - `backend/data/recorder/bybit_open_interest_1m/<SYMBOL>.jsonl`
3. Added merge utility in recorder module:
   - `mergeMinuteOiByTimestamp(...)`
   - extends existing timestamped rows only where `openInterestValue` is missing
   - does not replace existing boundary values
4. Connected recorder to WS ingestion flow:
   - `backend/src/api/wsHub.ts` now starts recorder in runtime stream lifecycle
   - ingests on each ticker update through dedicated recorder call
   - stops recorder when runtime streams stop
5. Extended process status output:
   - `backend/src/api/http.ts` now exposes recorder counters and state from the real recorder subsystem, replacing placeholder-only fields.
6. Frontend process type compatibility:
   - `frontend/src/shared/types/domain.ts` recorder status shape extended additively
   - `frontend/src/features/session/components/ProcessIndicatorsBar.tsx` shows recorder writes/symbol counts.

## 3) Tests added
1. New backend tests:
   - `backend/src/tests/minuteOiRecorder.test.ts`
2. Covered scenarios:
   - recorder skips 5-minute boundary points and writes only intermediate minute points
   - duplicate writes for same minute are prevented
   - timestamp-merge helper fills missing intermediate minute OI without replacing existing boundary rows
