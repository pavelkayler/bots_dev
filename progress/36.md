1) What changed
- Updated the window-close cadence seeding condition in `backend/src/optimizer/runner.ts` so `prevWindowOivClose` is initialized when `openInterestValue` is finite and `>= 0` (instead of strictly `> 0`).

2) Why (prevent window_seed deadlock)
- This prevents cadence from being stuck in `window_seed` when early windows have zero OIV (due to missing OI alignment or legitimate zeros). With zero allowed as a seed, later windows can advance into normal decision flow.

3) How to verify
- Build backend and frontend:
  - `npm --prefix backend run build`
  - `npm --prefix frontend run build`
- Run optimizer over a range where early OI is missing/zero, then confirm logs/results progress from `window_seed` into `decisionsNoRefs` and later `ok_*` outcomes on subsequent window closes.
