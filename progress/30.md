1) Changes per file
- `backend/src/optimizer/runner.ts`
  - Switched candidate loop to evaluated/attempts model (`evaluated` only increments when a candidate is actually simulated).
  - Enforced remember-negatives skip before simulation and kept skip logic independent from `excludeNegative` UI filtering.
  - Added checkpoint visibility fields: `skippedBlacklistedTotal` and `negativeSetSize`.
  - Kept blacklist persistence flow and updated deterministic candidate key payload to fixed-order JSON with `{ directionMode, optTfMin, sim, params }`.
- `frontend/src/shared/hooks/useInterval.ts`
  - Extended hook with optional `enabled` flag while keeping stable callback behavior.
- `frontend/src/pages/optimizer/OptimizerPage.tsx`
  - Unified polling cadence to `POLL_MS = 1000` and replaced duplicated `setInterval` effects with `useInterval`.
  - Converted loop status polling, single-job status polling, dataset-cache sync polling, dataset-history polling, job-history polling, and run clock updates to shared 1Hz cadence.
  - Removed dependency-driven interval recreation patterns by using stable interval wiring.
  - Updated UI copy: `tf (opt)` -> `signal window (min)` plus hint `Execution: close-only replay (no OHLC extrema).`
- `docs/11_current_state.md`
  - Updated optimizer mechanics for cache sources, close-only replay, OI/OIV handling, funding gating, and signal window terminology.
- `docs/17_optimizer.md`
  - Added data/replay mechanics section (1m price cache, 5m OI grid, funding history last-known application, close-only replay, fees vs funding fee policy).
  - Clarified `signal window (min)` naming and deterministic candidate-key scope for remember-negatives.
- `docs/04_paper_execution.md`
  - Added concise note aligning optimizer replay semantics (close-only ticks, funding gating only, no funding fee in optimizer pnl).

2) Why (A–D mapping)
- A) rememberNegatives correctness
  - Ensures blacklisted negatives are skipped pre-simulation on subsequent draws under same runKey.
  - Guarantees requested candidate count is achieved for non-blacklisted simulations (within attempts cap).
  - Exposes blacklist skip/set size in checkpoints for backend visibility.
- B) Polling centralization
  - Standardized polling to 1Hz with one shared hook pattern to eliminate duplicated intervals and dependency-triggered interval churn.
- C) Docs updates
  - Synced docs with current paper-like optimizer mechanics and current data-source behavior.
- D) Terminology clarity
  - Clarified signal window vs execution tick semantics in both UI and docs.

3) How to verify locally
- Build checks:
  - `npm --prefix backend run build`
  - `npm --prefix frontend run build`
- Functional checks:
  - Start optimizer loop with `rememberNegatives` ON, let one run finish with negative rows, then observe next run skipping previously negative candidate keys for same runKey.
  - Confirm loop checkpoints include `skippedBlacklistedTotal` and `negativeSetSize`.
  - Confirm optimizer page polling remains active at 1Hz and does not create duplicate requests when local UI state changes.
  - Confirm the label now shows `signal window (min)` with execution hint text.

4) Known limitations
- Attempts cap (`candidates * 50`) can stop early if blacklist density becomes extreme and unique candidates are hard to sample.
- Dataset/history/job polling is now uniformly 1Hz, which is intentionally more frequent than prior mixed cadences.
