# 4 — Dataset history export now exports movements

## Goal
Replace the dataset history **Export** button (previously exporting only metadata) with an export of **all cached kline movements** within the history time window.

## What changed
1. **Backend**: `/api/data/history/:id/export` now streams **NDJSON**:
   - first line: `{ type: "meta", history }`
   - subsequent lines: `{ type: "kline", row }` for each cached candle within `[startMs, endMs]` for each symbol in `receivedSymbols`
   - missing cache files produce `{ type: "warning", symbol, code: "cache_missing" }`
   - response includes `Content-Disposition` so the browser downloads as `history_<id>_data.jsonl`

2. **Frontend**: dataset history Export button now downloads the NDJSON blob and saves it as `history_<id>_data.jsonl`.

## Files changed
- `backend/src/api/http.ts`
- `frontend/src/features/datasetHistory/api/datasetHistoryApi.ts`
- `frontend/src/pages/optimizer/OptimizerPage.tsx`

## Notes
- This keeps **variant A** (metadata-only history storage). The export reads from `backend/data/cache/bybit_klines/*.jsonl` at download time.
- The export is streaming to avoid loading the whole dataset into memory.
