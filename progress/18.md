1) What changed per file
- `frontend/src/pages/optimizer/OptimizerPage.tsx`
  - Removed optimizer funding simulation storage key/state/effect/UI input.
  - Removed `sim.fundingBpsPer8h` from optimizer loop start payload.
  - Kept `directionMode` control and payload wiring unchanged.
- `frontend/src/features/optimizer/api/optimizerApi.ts`
  - Removed `fundingBpsPer8h` from `OptimizerSimulationParams`.
- `backend/src/api/http.ts`
  - Updated `parseSimParams(raw)` to stop parsing/validating/returning `fundingBpsPer8h`.
  - Kept sim parsing tolerant of extra fields (ignored).
- `backend/src/optimizer/runner.ts`
  - Removed funding cooldown gate usage from optimizer loop.
  - Set optimizer `SignalEngine` to `requireFundingSign: false`.
  - Passed neutral funding values into optimizer paper tick (`fundingRate: 0`, `nextFundingTime: 0`, `cooldownActive: false`).
  - Changed end-of-range handling to `paper.stopAll(..., closeOpenPositions: false)` so unfinished positions are discarded.
  - Reduced cache replay intrabar bias:
    - bull/flat candle: open -> low -> high -> close
    - bear candle: open -> high -> low -> close
  - Removed turnover/volume as OI fallback in cache replay; now only `oi * close` or `0`.
- `backend/src/paper/PaperBroker.ts`
  - Extended `stopAll` with optional `closeOpenPositions?: boolean` (default `true`).
  - When `closeOpenPositions: false`: pending orders are still canceled/logged, but open positions are cleared without force-close event/stat/PnL impact.
- `backend/src/dataset/receiveDataStore.ts`
  - Removed Bybit funding history fetch types/helpers/flow from Receive Data.
  - Removed per-candle funding alignment and stopped writing funding fields to cache rows.
  - Kept OI fetching/alignment and metadata as-is (`hasFunding` now persisted as `false`).

2) Why (mapping to goals A-D)
- A (remove funding from optimizer pipeline): removed optimizer funding input/type/parser and backend optimizer sim funding field.
- B (optimizer must not use funding for direction/cooldown): optimizer run now ignores funding sign and funding cooldown, and paper tick receives neutral funding values.
- C (exclude unfinished trades): optimizer stop path now discards open positions at range end (no force close accounting).
- D (remaining fixes): intrabar replay order now depends on candle direction to reduce path bias; OI fallback no longer uses turnover/volume proxies.

3) How to verify
- Build/typecheck:
  - `cd backend && npm run build`
  - `cd frontend && npm run build`
- UI smoke:
  - Open `/optimizer` and confirm there is no `fundingBpsPer8h` field in simulation controls.
  - Start loop with `directionMode=long`; verify result trade logs/counters show only LONG entries/closes.
  - Start loop with `directionMode=short`; verify result trade logs/counters show only SHORT entries/closes.
  - Use a dataset/range where some trades would stay open by end-of-range; verify `closesForce` stays 0 and `trades`/`netPnl` only reflect completed TP/SL closes.
- Replay behavior checks:
  - With `DEBUG_OPT_MARKETDATA=1`, run optimizer and inspect logs for cache interval/sample metrics.
  - Optionally inspect cached rows under `backend/data/cache/bybit_klines/<interval>/<symbol>.jsonl` to confirm funding fields are no longer written by Receive Data.

4) Known limitations kept intentionally (minimal diff)
- Did not redesign optimizer/result schemas beyond requested fields and behavior changes.
- Existing debug instrumentation was preserved and not expanded.
- DirectionMode runtime validation is covered by existing logic; no new test harness was added in this patch.
