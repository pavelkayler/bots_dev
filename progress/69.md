# 69 - Stage 10 Follow Tail mode (stability-first)

Date: 2026-03-06

## 1) Problem
Optimizer payload types already mentioned `datasetMode: "snapshot" | "followTail"`, but runtime behavior was still snapshot-only:
- backend resolver was a stub (`withDatasetResolved` returned payload as-is)
- no strict follow-tail window normalization (`timeRangeToTs` was not re-bound to now)
- no frontend mode controls for Follow Tail input

## 2) Backend changes
Files:
- `backend/src/api/http.ts`
- `backend/src/optimizer/loopStore.ts`

What changed:
1. Added `resolveOptimizerDatasetWindow(runPayload, nowMs?)` and used it inside `withDatasetResolved`.
2. Enforced dataset mode validity:
   - allowed: `snapshot`, `followTail`
   - invalid mode -> `invalid_dataset_mode`.
3. Follow Tail normalization:
   - requires valid `timeRangeFromTs`
   - sets `timeRangeToTs` to current time at run start
   - ignores stale prefilled `timeRangeToTs`.
4. Safety around loop scheduling:
   - `startLoopJob` now catches resolution errors and stops loop with bounded `lastError`, instead of throwing.
5. Added `datasetMode` into run key and stored history payload shape.
6. Extended `OptimizerLoopRunPayload` typing with dataset mode/time fields.

## 3) Frontend changes
Files:
- `frontend/src/pages/optimizer/OptimizerPage.tsx`
- `frontend/src/features/optimizer/api/optimizerApi.ts`
- `frontend/src/features/optimizer/utils/datasetWindow.ts` (new)

What changed:
1. Added Optimizer controls:
   - `Dataset mode`: `Snapshot` / `Follow Tail`
   - `Follow Tail start`: `datetime-local` input.
2. Persisted mode/start in localStorage.
3. Start Loop payload now includes:
   - `datasetMode`
   - `timeRangeFromTs` for Follow Tail
   - no fixed `timeRangeToTs` from UI (backend resolves it per run).
4. Extended frontend optimizer history payload type with optional `datasetMode`.

## 4) Tests added/updated
Backend:
- `backend/src/tests/optimizerFollowTail.test.ts` (new)
  - snapshot keeps static behavior
  - follow tail end is forced to `now`
  - invalid follow tail start is rejected

Frontend:
- `frontend/src/features/optimizer/utils/datasetWindow.test.ts` (new)
  - datetime-local format/parse
  - snapshot payload behavior
  - follow tail start validation
  - follow tail payload generation
