# Stage 4 - Recorder Modes

## 1) Goal
Enable two recorder operation modes on top of Stage 3 foundation:
1. `record_only`
2. `record_while_running`

## 2) Backend changes
1. Extended recorder mode model in `backend/src/recorder/MinuteOiRecorder.ts`:
   - mode enum now supports `off | record_only | record_while_running`
   - added explicit lifecycle methods:
     - `setMode(mode)`
     - `activate(symbols)`
     - `deactivate()`
2. Updated WS hub stream lifecycle in `backend/src/api/wsHub.ts`:
   - stream enable decision now includes recorder mode (`record_only` can keep streams alive without runtime session)
   - recorder activation/deactivation now follows current mode + runtime state
   - added `requestStreamLifecycleSync()` export for safe on-demand lifecycle resync
3. Added recorder control endpoints in `backend/src/api/http.ts`:
   - `GET /api/recorder/status`
   - `POST /api/recorder/mode` with strict mode validation
   - mode updates trigger immediate stream lifecycle resync

## 3) Behavior summary
- `record_only`:
  - recorder can run without active trading runtime
  - WS stream stays enabled for ticker ingestion
- `record_while_running`:
  - recorder activates only when runtime is RUNNING or RESUMING
  - recorder deactivates when runtime stops/pauses

## 4) Tests updated
1. Extended `backend/src/tests/minuteOiRecorder.test.ts` with mode behavior test:
   - verifies writes in `record_only`
   - verifies no writes while `record_while_running` is inactive
   - verifies writes resume when re-activated
