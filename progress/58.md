# Progress Report 58

## 1) Current issue package
1. Receive Data could fail early with `incomplete minute OI coverage` even when CoinGlass state was only temporary rate-limit waiting.
2. CoinGlass execution was not in the requested sequential handoff mode.
3. Receive Data progress had no ETA.
4. Optimizer `optTfMin` path was restricted to 15+.
5. Higher-timeframe OI reference used a coarse single-point behavior instead of minute-path aggregation.

## 2) Why false failure happened
1. CoinGlass flow previously ran inside the same per-symbol Bybit loop and did not separate retryable waiting from terminal coverage failure semantics.
2. Final missing-minute check could be hit in the same pass where CoinGlass had only temporarily waited.

## 3) New scheduling model (sequential handoff)
1. Added deterministic handoff helper in `backend/src/dataset/receiveDataStore.ts`:
   - `SequentialCoinGlassHandoff`
2. Runtime behavior now:
   - Bybit fully processes symbol K
   - when Bybit advances to symbol K+1, CoinGlass processes only symbol K
   - queue is bounded to one pending symbol
   - after final Bybit symbol, CoinGlass drains the pending symbol before completion

## 4) Waiting/retry behavior and terminal failure rules
1. Added explicit CoinGlass stage semantics in `backend/src/dataset/receiveDataStore.ts`:
   - `idle | running | waiting_rate_limit | retrying | failed | completed`
2. Retryable behavior:
   - `CoinGlassRateLimitError` is treated as retryable waiting
   - job remains alive and continues after sleep/reset window
3. Terminal behavior:
   - final `dataset_quality_error` emitted only when CoinGlass stage finished attempts and minute OI still missing
   - `COINGLASS_API_KEY` missing and unsupported symbol mapping remain terminal with clear error codes

## 5) ETA logic
1. Added backend ETA computation in `backend/src/dataset/receiveDataStore.ts`:
   - `estimateReceiveEtaSec(...)`
2. ETA model includes:
   - completed/total throughput
   - explicit CoinGlass wait-until delay
3. Progress payload is additive (`etaSec`), and UI renders it via:
   - `frontend/src/features/datasetTarget/ui/receiveProgress.ts`
   - format `ETA: ~Xm Ys`

## 6) Optimizer timeframe expansion
1. Backend validation in `backend/src/api/http.ts` now allows `optTfMin` from 5..240 (was 15..240).
2. Runner minimum tf lowered to 5 in `backend/src/optimizer/runner.ts`.
3. Frontend selector now supports `5, 10, 15, 30, 60, 120, 240`:
   - `frontend/src/features/optimizer/utils/timeframes.ts`
   - `frontend/src/pages/optimizer/OptimizerPage.tsx`

## 7) Minute-level OI precision change in optimizer
1. Added minute-path window aggregation helper in `backend/src/optimizer/runner.ts`:
   - `deriveWindowOiValue(...)`
2. For higher signal windows, OI move references now use aggregated minute OI path collected inside each window, not only a single coarse boundary value.
