# 79 - Signal Bot isolation finish (semantics, preset scope, safety)

1. What remained broken
- Signal/OI optimizer semantics were mostly separated, but there were still UX-level leaks:
  - optimizer legends still contained OI-style generic text in mixed places
  - pagination/page-size storage used redundant key layering and could produce noisy storage keys
- Bot preset isolation needed one more explicit regression guard test to prevent cross-bot preset listing regressions.

2. Optimizer storage hardening
- Updated `/frontend/src/pages/optimizer/OptimizerPage.tsx`:
  - removed redundant `*.pageSize` scoped keys in local state map
  - switched page-size persistence hooks to table ids directly:
    - dataset history -> `storageKeys.datasetHistoryTableId`
    - results -> `storageKeys.resultsTableId`
    - run history -> `storageKeys.jobHistoryTableId`
- This keeps page-size persistence bot-scoped while reducing key bloat and lowering localStorage pressure risk.

3. Signal/OI result semantics cleanup
- Updated `/frontend/src/pages/optimizer/OptimizerPage.tsx`:

- Signal column labels remain mapped by bot through `getRangeLabel(...)`.
