1) What changed per file
- `backend/src/optimizer/runner.ts`
  - Added per-row long/short aggregates to optimizer results (`longsCount`, `longsPnl`, `shortsCount`, `shortsPnl`).
  - Added deterministic `buildCandidateKey(...)` including direction mode, opt timeframe, sim fill settings, and strategy params.
  - Switched remember-negatives skip/write logic to use `candidateKey` instead of param-only signature.
  - During candidate simulation, now accumulates long/short closed TP/SL counts and realized pnl totals.
- `frontend/src/features/optimizer/api/optimizerApi.ts`
  - Extended `OptimizationResult` with long/short aggregate fields.
  - Extended `OptimizerSortKeyExtended` with `longsCount`, `longsPnl`, `shortsCount`, `shortsPnl`, and `direction`.
- `frontend/src/pages/optimizer/OptimizerPage.tsx`
  - Renamed row action button label from `Export trades` to `Export`.
  - Removed `expectancy/profitFactor/maxDD` columns from the main results table and added `Longs` and `Shorts` columns (`count / pnl`).
  - Added text-aware sorting support (`direction`) and numeric sorting for new long/short keys.
  - Updated header sort interactions and default sort direction logic (text defaults asc, numeric defaults desc).
  - Kept `rememberNegatives` payload wiring unchanged and display-only hide-negative behavior unchanged.

2) Why (map to items 1–4)
- (1) Updated action label per request (`Export trades` -> `Export`).
- (2) Fixed remember-negatives across runs by using a stable candidate key that matches repeated candidates and is persisted in blacklist state for subsequent runs in the same runKey scope.
- (3) Simplified main table by removing requested derived stats columns and adding actionable long/short outcome columns.
- (4) Added lightweight client-side sorting support for the requested columns and direction text, including live loop updates.

3) How to verify locally
- Build:
  - `npm --prefix backend run build`
  - `npm --prefix frontend run build`
- Functional checks:
  - Start optimizer loop with `Remember negatives` ON; confirm negative candidates from run N are skipped in run N+1.
  - Toggle `Hide negative netPnl` during run; confirm it only hides/shows displayed rows and does not affect remember-negatives behavior.
  - Confirm main results table headers sort rows (netPnl, trades, winRate, direction, longs/shorts, execution counters, params).
  - Confirm Longs/Shorts columns show `count / pnl` and action button label is `Export`.

4) Known limitations
- Remember-negatives persistence remains scoped by existing blacklist/runKey mechanics (file-backed blacklist by runKey); behavior is session/job compatible without introducing a new persistence layer.
- Sorting is client-side for loop display and lightweight by design.
