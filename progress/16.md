# Progress Report 16

1) Root cause identified
- Optimizer Receive Data cache replay emitted one `ticker` per candle using only `close`, then `kline_confirm`.
- With limit entries (`entryOffsetPct`) and ROI-based TP/SL checks, intra-candle low/high touches were never presented to PaperBroker, so many candidates ended with `trades=0`.

2) Backend timeframe selection for mixed history intervals
- Updated optimizer run/loop start handling to choose a deterministic maximum interval when selected history rows contain mixed intervals.
- Ordering used: `1 < 3 < 5 < 15 < 30 < 60 < 120 < 240 < 360 < 720 < D < W < M`.
- Behavior change: no interval mismatch error for mixed selections.
- Minimal handling applied: only histories matching the chosen max interval are used for `cacheDatasets`; smaller-interval histories are ignored (no resampling performed).
- Cache existence checks now run against the chosen interval path.

3) Receive Data cache replay now emits intra-candle ticks (cache mode only)
- In optimizer cache branch, each candle now produces synthetic ticker ticks in deterministic Variant A order:
  - `open -> low -> high -> close`
- For no-range candles (`high <= low`), replay emits a single ticker tick at `close`.
- `kline_confirm` remains emitted exactly once per candle at close and after the ticker ticks.
- This change is limited to the Receive Data cache optimizer path and does not alter live/paper runtime logic.

4) Backend DEV-only per-run summary logging
- Added `DEBUG_OPT_TRADES=1` guarded log output once per optimizer run with:
  - selected interval,
  - sample symbol,
  - candle count for sample symbol,
  - `placedOrders`, `filledOrders`, `closedTrades` aggregate counters.

5) Frontend DEV-only dataset timeframe debug logging
- Added deterministic interval ordering helper mirroring backend.
- On optimizer loop start, if `import.meta.env.DEV` and `localStorage.debugDatasetTf === "1"`, console logs:
  - selected history count,
  - intervals present in selected rows,
  - chosen max interval.

6) Previous attempts notes
- Initial patch attempt used a shell-level patch flow that failed partially due context mismatch while editing `runner.ts`.
- Re-applied changes with targeted file edits and validated resulting source sections before continuing.
