# Progress 3 — Dataset History (Receive Data → selectable histories for Optimizer Loop)

## 1) Goal
Implement “Dataset history” (Receive Data snapshots) with:
- server-side history storage (metadata only; data remains in `data/cache/bybit_klines/*.jsonl`)
- overwrite-last behavior for identical `(universeId, from, to)`
- frontend table with sorting + pagination (10 rows/page) and checkbox selection
- optimizer loop start requires selecting 1+ history rows, and runs on the combined time windows (oldest → newest)
- per-row actions: Delete, Export (JSON)

## 2) What was implemented

### 2.1 Backend: history storage (metadata only)
- New folder: `backend/data/history/`
- New index: `backend/data/history/index.json`
- New per-history meta: `backend/data/history/<jobId>/meta.json`

Overwrite-last:
- if the most recent history has the same paramsKey `universeId|startMs|endMs`,
  it is removed and replaced by the newly completed Receive Data job.
- loopsCount is carried over to the new record.

Symbols count:
- history stores only successfully downloaded symbols (`receivedSymbolsCount`).

### 2.2 Backend: Receive Data writes history record on DONE
- On Receive Data completion, backend writes/updates a history record (best effort).

### 2.3 Backend: History HTTP API
- `GET /api/data/history` → list histories
- `GET /api/data/history/:id/export` → export single history meta JSON
- `DELETE /api/data/history/:id` → delete history (index + folder)

### 2.4 Backend: Optimizer Loop uses datasetHistoryIds
- `POST /api/optimizer/loop/start` now requires `datasetHistoryIds: string[]`.
- Backend resolves each history meta, builds `cacheDatasets[]` sorted by startMs,
  validates that cache files exist for all referenced symbols,
  increments loopsCount for each selected history (delta=1).
- Payload now passes `cacheDatasets` to the optimizer worker/runner.

### 2.5 Backend: Runner supports multi-window cacheDatasets
- Runner accepts `cacheDatasets` and builds per-symbol merged time windows.
- Candles are included if they fall into ANY selected window for that symbol.
- This allows combining multiple history rows (oldest → newest by timestamp).

### 2.6 Frontend: Dataset history table + selection gating
- New table on `/optimizer` under Receive Data:
  - columns: from, to, range (actual duration), universe(name + receivedSymbolsCount),
    received, loop runs
  - checkbox selection on the left
  - actions: Export, Delete
  - sort by any column (click header)
  - pagination: 10 rows/page
- Start loop button is disabled until at least one history row is selected.
- Starting loop sends `datasetHistoryIds: selectedHistoryIds` to backend.

## 4) Notes / edge cases
- History is metadata-only; deleting history does NOT delete `data/cache/bybit_klines/*.jsonl`
  (this is intended for variant A).
- loopsCount counts loop-start actions (one increment per “Start loop” click),
  not internal runsCount within a loop.
