# Progress Report 57

## 1) Problem summary
1. Receive Data used Bybit 1m kline plus Bybit 5m OI and did not enforce strict minute OI completeness.
2. Optimizer history table assumed `row.runPayload.datasetHistoryIds` always exists and called `.join(...)`, causing UI crash on legacy/malformed rows.
3. React Router warning for `v7_startTransition` persisted because `RouterProvider` was not opted in.
4. Frontend had no focused regression tests for these touched areas.

## 2) CoinGlass integration design
1. Bybit stays primary and authoritative.
2. Runtime/live path is unchanged and remains Bybit WS only.
3. CoinGlass is used only in Receive Data historical pipeline for Bybit 1m OI minute gap-fill.
4. Gap-fill policy:
   - preserve Bybit OI values where Bybit OI timestamp exists (especially 5m boundaries)
   - fill only in-between minute slots from CoinGlass
   - do not use CoinGlass for prices/funding/other fields
5. Completion policy is strict:
   - Receive Data job is `done` only when all required 1m candle rows have OI
   - missing minute OI causes clear `dataset_quality_error`
6. CoinGlass limiter policy:
   - 30 req/min window (Hobbyist plan)
   - wait and retry on saturation
   - progress message includes countdown (`CoinGlass limit resets in N sec`)
