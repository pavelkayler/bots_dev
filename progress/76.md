# 76 - Multi-bot isolation hardening (optimizer/config/history/state)

Date: 2026-03-08

## 1) Problem summary
Observed leaks between `oi-momentum-v1` and `signal-multi-factor-v1`:
- mixed optimizer history/results in UI
- shared optimizer localStorage state
- fragile settings coupling through global selection side-effects
- incomplete bot scoping for "current optimizer job" API paths

## 2) Backend fixes

### 2.1 Bot-scoped optimizer history filtering
Files:
- `backend/src/api/http.ts`
- `backend/src/tests/optimizerHistoryAggregation.test.ts`

Changes:
- Added `filterOptimizerHistoryByBot(records, requestedBotId)`.
- `/api/optimizer/jobs/history` now accepts `botId` and filters by persisted `runPayload.selectedBotId`.
- Legacy rows without bot id are treated as default OI compatibility rows only.

### 2.2 Bot-scoped current optimizer job resolution
File:
- `backend/src/api/http.ts`

Changes:
- Added `resolveOptimizerJobBotId(job)`.
- `resolveCurrentOptimizerJobId(requestedBotId?)` now supports bot-scoped resolution.
- `/api/optimizer/jobs/current` accepts `botId`.
- `/api/optimizer/jobs/current/export` accepts `botId`.

### 2.3 Bot registry config behavior split
Files:
- `backend/src/bots/registry.ts`
- `backend/src/tests/botRegistry.test.ts`

Changes:
- Signal bot now has dedicated normalize/validate functions:
  - `normalizeSignalBotConfig`
  - `validateSignalBotConfig`
- Registry no longer points Signal bot directly to OI normalize/validate handlers.

## 3) Frontend fixes

### 3.1 Bot-scoped optimizer API consumption
Files:
- `frontend/src/features/optimizer/api/optimizerApi.ts`
- `frontend/src/features/optimizer/api/optimizerApi.test.ts`
- `frontend/src/pages/optimizer/OptimizerPage.tsx`

Changes:
- Optimizer history request carries `botId`.
- Current-job request/export carries `botId`.
- Optimizer page uses active/forced bot id for history and current-job flows.

### 3.2 Bot-scoped optimizer persistence keys
Files:
- `frontend/src/features/optimizer/utils/storageScope.ts`
- `frontend/src/features/optimizer/utils/storageScope.test.ts`
- `frontend/src/pages/optimizer/OptimizerPage.tsx`
- `frontend/src/shared/ui/TablePaginationControls.tsx`

Changes:
- Added scoped key helpers:
  - `resolveOptimizerStorageBotId`
  - `makeOptimizerScopedStorageKey`
- All optimizer local state now persists under bot-scoped keys.
- Legacy global keys are used as one-time fallback for backward compatibility.
- Pagination storage now reloads when table key changes.
- Dataset history/results/job-history pagination table ids and page-size keys are bot-scoped.

### 3.3 Signal page cross-coupling reduction
File:
- `frontend/src/pages/signalBot/SignalBotPage.tsx`

Changes:
- Removed forced global `selectedBotId` mutation side-effect on page open.
- Signal page keeps bot scope via forced optimizer id/settings path without changing global selector state unexpectedly.
