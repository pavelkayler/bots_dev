# 78 - Signal Bot optimizer semantics and preset patch isolation

1. Problem observed
- Signal Bot optimizer `Copy` still produced OI-style pending patches (`signals.priceThresholdPct`, `paper.entryTimeoutSec`, `paper.rearmSec`) and wrote to a global key.
- This allowed cross-bot config leakage and made Signal Bot copy/apply semantics inconsistent with its own parameter model.
- Signal optimizer range defaults were still OI-oriented and not aligned with overheating/exhaustion tuning.

2. Frontend optimizer fixes
- Updated `/frontend/src/pages/optimizer/OptimizerPage.tsx`:
  - Added bot-specific range defaults:
    - OI defaults kept as-is.
    - Signal defaults switched to signal-model ranges:
      - `priceTh` 0.1..3
      - `oivTh` 0.1..6
      - `tp` (CVD threshold) 0..3
      - `sl` (funding abs) 0..0.005
      - `offset` (min bars) 0..20
      - `timeoutSec` (lookback candles) 1..24
      - `rearmSec` (cooldown candles) 0..24
  - `loadSavedRanges` now accepts defaults and uses bot-aware defaults.
  - Removed Signal-only payload override that forcibly replaced `tp/sl/offset` with OI defaults.
  - Reworked `copyToSettings`:
    - Signal bot now stores bot-specific patch payload as:
      - `patch.botConfig.signals.priceMovePct`
      - `patch.botConfig.signals.oiMovePct`
      - `patch.botConfig.signals.cvdMoveThreshold`
      - `patch.botConfig.signals.fundingMinAbsPct`
      - `patch.botConfig.signals.minBarsBetweenSignals`
      - `patch.botConfig.strategy.lookbackCandles`
      - `patch.botConfig.strategy.cooldownCandles`
    - OI bot keeps OI patch shape.
    - Pending patch key is now bot-scoped: `bots_dev.pendingConfigPatch.<botId>`.

3. Settings patch-consumption isolation
- Updated `/frontend/src/features/config/components/ConfigPanel.tsx`:
  - Reads bot-scoped pending patch key first and legacy key second.
  - Applies patch only when `patch.botId` matches active bot.
  - Removes only the key that was consumed.
- Updated `/frontend/src/pages/signalBot/SignalBotSettingsPanel.tsx`:
  - Added pending patch consume path for Signal Bot.
  - Merges `patch.botConfig.signals` and `patch.botConfig.strategy` into draft.
  - Persists merged form and clears consumed pending patch key.

4. Tests added
- Updated `/backend/src/tests/signalEngine.test.ts`:
  - Added test for `requireFundingExtreme + fundingMinAbsPct` blocking weak-funding signals.
  - Added test for `requireCvdDivergence` enforcement and positive path with valid divergence.
