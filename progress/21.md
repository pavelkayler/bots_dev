1) What changed per file
- `frontend/src/pages/optimizer/OptimizerPage.tsx`
  - Updated optimizer TF localStorage restore/save normalization to require `>= 15`, with invalid/missing values coerced and persisted as `"15"`.
  - Removed sub-15 TF options from `tf (opt)` and left only `15/30/60/120/240`.
- `backend/src/api/http.ts`
  - Tightened `/api/optimizer/run` validation to reject `optTfMin < 15`.
  - Tightened `/api/optimizer/loop/start` validation to reject `optTfMin < 15`.
- `backend/src/optimizer/runner.ts`
  - Reworked cacheDataset replay to emit close-only ticker events at candle close for all replay intervals.
  - Kept `kline_confirm` emission at close timestamp for downstream consistency.
  - Ensured replay ticker payload uses only real OI (`oi * close` if present), with `openInterestValue` and `openInterest` set from that value, and `fundingRate: 0` always.
  - Added per-tape ticker interval sampling in cacheDataset replay and pushed tape median intervals into `globalTickIntervals` (bounded by `MAX_TICK_INTERVAL_SAMPLES`).
  - Changed timeout adaptation to `max(baseTimeout, ceil(medianTickIntervalSec) + 1, 5)`.
  - Enforced rearm floor per tape timeframe via `minRearmMs = effectiveTfMin * 60_000` and `rearmDelayMs = max(floor(params.rearmMs), minRearmMs)`.

2) Why
- This locks optimizer behavior into paper-like constraints and avoids replay-side edge:
  - TF floor `>= 15m` prevents hyper-reactive signal windows not intended for paper parity.
  - Close-only replay removes synthetic intrabar extrema touches from OHLC paths.
  - Funding remains ignored (`0`) and OI is not proxy-filled from volume/turnover, reducing synthetic data leakage.
  - Tick-interval-derived timeout now stays next-tick-safe without the previous 2x inflation.
  - Rearm delay now matches at least one effective signal timeframe window to reduce overly frequent retries.

3) How to verify locally
- Run builds:
  - `npm --prefix backend run build`
  - `npm --prefix frontend run build`
- API checks:
  - POST `/api/optimizer/run` with `optTfMin=5` should return 400; `optTfMin=15` should pass validation.
  - POST `/api/optimizer/loop/start` with `optTfMin=5` should return 400; `optTfMin=15` should pass validation.
- Replay behavior checks:
  - Run optimizer with cacheDataset at interval `1` and `15`; verify one ticker per candle at close and a close-time `kline_confirm`.
  - Verify ticker payload funding is always `0` and OI/OIV uses only `row.oi` when present.
  - Verify entry orders do not expire before next observed close tick in close-only mode and rearm cadence is not faster than `optTfMin`.

4) Known limitations intentionally kept
- Close-only replay intentionally lowers fill opportunities versus synthetic OHLC pathing.
- OI remains stepwise from aligned cache granularity (commonly 5m), so intra-step OI dynamics are not reconstructed.
- Optimizer intentionally ignores funding effects by design in this mode.
